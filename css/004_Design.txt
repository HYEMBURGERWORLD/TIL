웹디자인 vs 웹개발
웹디자인 > 웹사이트 전반적인 모습과 느낌
웹개발 > css, js를 통해 디자인을 구현

예전에는 디자이너 -> 개발자 -> 사용자 라는 느낌으로 구분지어졌지만
요즘에는 개발하는 디자이너, 디자인하는 개발자가 점점 많아지는 추세로
디자인 하는 개발자를 만드는 게 이 강의의 목표 

디자인 프로그램 없이
코드로만 웹사이트 작성, 빌드, 디자인할 수 있도록 실력을 기르는 게 웅앵

왜 디자인에 신경써야할까?
1. 사용자에게 있어 디자인 뒤에 있는 브랜드나 제품에 즉시 좋은 이미지를 심어줄 수 있다.
만약 디자인을 신경쓰지 않는다면 사용자가 생각하기에 아 이 브랜드는 자기가 제공하는 상품, 서비스에 별 관심이 없구나 라는 인상을 심어줄 수 있음. 
2. 사용자가 바로 신뢰할 수 있게 함
반대 > 신뢰를 불안하게 함
3. 사용자가 방문하는 브랜드나 상품에 인식 가치를 높여준다
반대 > 싸구려로 보이게 함
4. 사용자가 이 사이트에 들어오고자 했던 목적을 바로 찾을 수 있게 하는 게 좋은 디자인!

1. 타이포그래피
활자를 배치하는 기술로 잘 읽힐 때 인상 업
글자의 아름다움을 살리고 읽기 쉽게 만드는 게 중요하다

세리프 / 산세리프
전통적 고전적 형태 인터페이스. 신뢰성 전달.

산세리프
깔끔. 단순. 현대적

- 좋고 인기 있는 폰트를 사용한다 (안전!)
- 폰트는 한 웹페이지에 여러개 쓸 필요x 오히려 어지러워 보임. 한 개 혹은 두 개 정도 사용.
- 웹사이트 개성이나 성격에 맞는 폰트를 사용한다 (개성고르기 -> 세리프/산세리프 고르기)
- 웹페이지의 메세지를 가장 잘 전달할 폰트를 골라보자!
- 크기를 고를 때는 선택범위를 제한해서 그 안에서 고르자 。
- 일반 텍스트의 크기는 16-32픽셀 안이어야 한다. 
- 긴 텍스트의 경우 20픽셀 혹은 그것보다 조금 더 크게 
- 헤드라인 같은 것에 50픽셀 이상 font-weight 값 600이상도 사용할 수 있다. 
- 폰트의 굵기를 400이하로 하지말 것. 읽기 힘들어짐. 
- 한줄당 75자 이하
- 줄높이 1.5 ~ 2
- 헤드라인이 부자연스러워보인다면 글자사이를 줄여보자. (-3.5 이런식으로)
- 짧은 제목은 모두 대문자로. 굵게, 글자 간격 넓히기
- 양쪽정렬 하지말것(충격그자체임)
- 긴 텍스트 블록을 가운데 정렬하지 말 것

5. 그림자

디자인 각 부분간의 관계를 파악하는데 도움을 준다.
흥미로운 시각적 디테일 추가

기본적으로 사용자 인터페이스에서 깊이를 시뮬레이션 한다.
다시 말해 디자인에 3차원을 추가한다. (사용하는 이유)

박스, 텍스트 모두 그림자를 줄 수 있다.

- 개성과 맞지 않으면 그림자 사용 필요 없음
- 조금씩 그림자를 써야 한다. (모든 요소에 넣지마!! 모든 요소에 넣으면 계층이 생기지 않음 결국 깊이감이 없다)
- 너무 어둡게 하지 않는다. 
- 작은 요소에는 작은 그림자를. 강조효과.
- 중간크기 그림자 넓은 면적. 섹션이 완전 띄워져잇는 느낌을 줘서 강조. 눈에 띄게 함. 
- 큰그림자. 전체에서 사용하려는 것. 팝업, 탐색 창 등. 가장 위에 떠있을 필요가 있는 것들. 
- 언제나 그림자가 검은색일 필요는 없다. 비슷한 색상을 줘서 빛나보이게 할 수 있다!!

7. white space 공백 
- 요소 사이의 공간
- 깔끔 현대적 세련적 이해가 더 편하게 됨
- 웹페이지내의 서로 다른 정보들이 어떻게 연관되어 있는지 자동으로 알게 해준다.
- 요소들 사이에 보이지 않는 관계를 만들어내기 때문에.

- 섹션 사이에 흰색 간격을 많이 남겨주자
- 요소들 사이에 있는 부분에도 사용하자 섹션 사이만큼은 아님 덕분에 명확히 윤곽을 잡을 수 있따!
- 구성 요소들 안에는 선보다는 여백을 넣어주는 게 더 좋다 

- 기본적으로 함께 속한 요소는 더 가까이 있어야 한다 다른 요소와의 거리보다. (근접성 법칙? Law of Proximity) 다른것과 무관하다는 걸 사용자가 알 수 있도록.
- 일단 흰색 공간(여백-공백)을 많이 채운다음 제대로 남을 때까지 점점 공간을 줄이는 방법!
- 다른 디자인과 맞춰보자. 텍스트나 아이콘이 크면 그만큼 여백은 더 필요하다. 
- 


8. visual hierarchy 시각적 계층구조(?)
- 특정 디자인의 어떤 요소가 가장 중요한지 정하는 것.
- 주의를 끌기 위해.
- - 사용자를 위해 경로를 정의하는 것. 직관적이고 자동적인 방법으로 이끌기 위해
- 의미있는 계층구조를 만들기 위해 저 위의 것들을 잘 조합하고 사용하는것!! 

- 중요한 요소를 페이지 상단에 배치하자
- 관심은 상단에서 하단으로 흐른다
- 이미지는 많은 관심을 끌기 때문에 주의해서 사용한다. 텍스트를 읽기 원한다면 이미지 사용 하지않거나 작은 이미지만을 사용한다.
- 백색 공간을 효과적으로 사용할것(계층구조만들기!)

텍스트의 경우
- 글꼴 크기, 두께, 색상(텍스트색상 / 배경색상), 여백으로 중요도를 정할 수 있다
- 제목, 서브제목, 링크, 버튼, 아이콘 조합

요소 간
- 배경색상, 그림자, 테두리 조합
- 추천서-광고 / 호출-액션 섹션 / 하이라이트 섹션 / 프리뷰 카드 / 양식 / 가격표 / 행이나 열 테이블의 중요한 정보를 강조

사용자들이 이를 사용하면서 달성하고자 했던 목표에 실제로 도달했는지 
… 등

둘이 다른 것처럼 보이지만 사실은 ui 디자인 없이는 ux 디자인은 존재할 수 없다.
ux 디자인에 ui 디자인이 많은 부분을 차지하고 있기 때문이다.

페이지를 방문하는 목적, 달성하려는 목표 … (보통 원하는 물건을 찾거나 구매하려고 하는) 

- 복잡한 레이아웃을 생성하지 않는다.
- 독창성 추구보다는 익숙한 패턴을 사용한다. 
- 콜투액션. 사용자가 더 빠르고 쉽게 목표에 도달할 수 있도록.
- 링크에만 파란색글씨+밑줄!! 다른 텍스트요소에 쓰면 링크라고 자연스럽게 착각할 수 있다. 
- 애니메이션은 200-500ms 사이의 길이로 넣을 수 있다고 마구잡이로 넣으면 전문성 down 으로 보일 수 있다.
- 양식을 만들 때는 작성이 아주 쉽도록 만들어야 한다. 세로 줄 한 줄에 필드 돠 돠 돳.
- 양식이 제출된 후 일어나는 일에 대해 인터페이스에 명확히 나와야 한다. (예를들어 이메일이 발송되었습니다. 같은 알림같은)
- 특정 행동을 위한 버튼이 있을 경우, 실제로 효과가 생성되는 곳에 있어야 한다. (예를들어 목록 추가되는 곳에 목록 추가 버튼을 놓는 것)
